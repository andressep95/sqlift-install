SQLift Documentation

Core Functionality

  • Automatically generates Java entity classes from SQL schema files
  • Converts SQL table definitions into JPA-annotated Java classes
  • Transforms database relationships into proper Java entity references
  • Supports both Jakarta EE and Java EE (javax) persistence annotations
  • Optional Lombok integration for boilerplate code reduction

Key Features

1. Schema Processing

  • Parses CREATE TABLE statements from SQL files
  • Handles complex table relationships (foreign keys)
  • Supports PostgreSQL data types (with MySQL, Oracle, and SQL Server planned)
  • Maintains table relationships in generated code
  • Smart naming convention conversion:
    • snake_case to camelCase for fields
    • Plural to singular for class names
    • Proper Java naming conventions

2. Code Generation

  • Generates clean, well-formatted Java code
  • Creates proper entity classes with:
    • Appropriate JPA annotations (@Entity, @Table, etc.)
    • Primary key handling (@Id, @GeneratedValue)
    • Foreign key relationships
    • Column mappings with constraints (@Column with name)
  • Includes:
    • Default constructor
    • All-args constructor
    • Getters and setters
    • Proper imports management

3. Configuration

  • YAML-based configuration file
  • Database engine selection
  • Schema file path specification
  • Output package configuration
  • JPA implementation choice (Jakarta/javax)
  • Lombok integration toggle
  • Flexible output options
  • Customizable package structure

Configuration

Configuration File

SQLift uses a YAML configuration file (sqlift.yaml) to customize its behavior. Here's a complete reference of available options:

Basic Configuration

version: "1"
sql:
  engine: "postgresql"  # Database engine
  schema: "schema.sql" # Path to schema file
  output:
    package: "com.example.entity"
    directory: "src/main/java"

Advanced Options

version: "1"
sql:
  engine: "postgresql"
  schema: "schema.sql"
  output:
    package: "com.example.entity"
    directory: "src/main/java"
    options:
      lombok: true  # Enable Lombok annotations
      jpa:
        enabled: true
        type: "jakarta"  # or "javax"
      naming:
        class:
          strategy: "singular"  # Convert plural to singular
        field:
          strategy: "camelCase"  # Convert snake_case to camelCase
      generation:
        constructors:
          default: true
          allArgs: true
        methods:
          getters: true
          setters: true
          toString: true
          equals: true
          hashCode: true

Configuration Options

Database Engine

Supported values for sql.engine:

  • postgresql (fully supported)
  • mysql (planned)
  • oracle (planned)
  • sqlserver (planned)

JPA Implementation

Choose between Jakarta EE and Java EE annotations:

  • jakarta - Uses jakarta.persistence.* packages
  • javax - Uses javax.persistence.* packages

Lombok Integration

When lombok: true is set, SQLift will:

  • Add @Data annotation for getters/setters
  • Use @NoArgsConstructor for default constructor
  • Use @AllArgsConstructor for all-args constructor
  • Add @Builder for builder pattern support

Environment Variables

SQLift also supports configuration through environment variables:

  • SQLIFT_CONFIG - Path to configuration file
  • SQLIFT_SCHEMA - Path to schema file
  • SQLIFT_OUTPUT_DIR - Output directory
  • SQLIFT_PACKAGE - Base package name
